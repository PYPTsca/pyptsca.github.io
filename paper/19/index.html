    三、计算题（每小题3分，共15分）

    1. 求十进制数（-16）的8位补码

    解题步骤：

    1. 求绝对值的原码：16的二进制是10000，8位原码为00010000（符号位0表示正数）。

    2. 求负数的原码：符号位改为1，其余位不变，得10010000。

    3. 求补码：负数补码 = 原码除符号位外取反加1

      • 除符号位：0010000 → 取反 → 1101111

      • 加1 → 1110000

      • 加符号位 → 最终补码11110000。

    答案：11110000

    2. 十进制数（78）₁₀转换成十六进制

    解题步骤：

    1. 十六进制基数为16，反复除以16取余数：

      • 78 ÷ 16 = 4 余 14（十六进制中14用E表示）

      • 4 ÷ 16 = 0 余 4

    2. 从下往上取余数，得4E。

    答案：4EH（或4E）

    3. 十进制数45.12转换成二进制（保留4位小数）

    解题步骤：

    1. 整数部分（45）转换：除以2取余，从下往上排列45→22余1→11余0→5余1→2余1→1余0→0余1 → 整数部分101101。

    2. 小数部分（0.12）转换：乘2取整，保留4位小数0.12×2=0.24（取0）→0.24×2=0.48（取0）→0.48×2=0.96（取0）→0.96×2=1.92（取1） → 小数部分0001。

    3. 组合结果：101101.0001。

    答案：101101.0001

    4. 计算表达式1>3 or 4>2 and False的值

    解题步骤：

    1. 逻辑运算优先级：and高于or，先算and部分。

    2. 4>2为True，True and False=False。

    3. 再算or：1>3为False，False or False=False。

    答案：False

    5. 计算表达式4//5%2的值

    解题步骤：

    1. 运算优先级：//（整除）和%（取余）同级，从左到右计算。

    2. 4//5=0（整除结果取商的整数部分）。

    3. 0%2=0（0除以2余数为0）。

    答案：0

    四、简答题（每小题5分，共25分）

    1. 算法的五个特征

    1. 有穷性：算法执行有限步骤后必须终止，不能无限循环。

    2. 确定性：每一步操作无歧义，给定输入只能得到唯一输出。

    3. 可行性：每一步都能通过基本运算（如加减乘除）有限次完成。

    4. 输入：0个或多个输入（0个输入指初始条件固定）。

    5. 输出：1个或多个输出（输出是问题的解，与输入对应）。

    2. 函数调用的好处

    1. 代码复用：封装重复功能，避免冗余（如多次计算阶乘只需调用一次函数）。

    2. 模块化设计：将复杂程序分解为独立功能模块，便于理解、维护和调试。

    3. 提高可读性：函数名直观体现功能（如fac(n)表示求阶乘），程序结构清晰。

    4. 简化修改：功能修改只需改函数内部，无需修改所有调用处。

    5. 灵活扩展：通过参数传递实现同一功能的不同场景应用（如不同n的阶乘）。

    3. 现代操作系统的中断类型及举例

    （1）硬件中断（外部中断）

    • 定义：由硬件设备触发，向CPU请求处理。

    • 举例：键盘输入中断（按键盘键）、鼠标中断、磁盘I/O中断（磁盘读写完成）、时钟中断（定时触发）。

    （2）软件中断（内部中断/异常）

    • 定义：由程序执行错误或主动请求触发。

    • 举例：除0异常、内存访问越界、系统调用（如printf底层调用）、断点中断（调试时）。

    4. 通用计算机的基本组成部分（冯·诺依曼体系）

    1. 运算器：负责算术运算（加减乘除）和逻辑运算（与或非）。

    2. 控制器：协调各部件工作，执行“取指-译码-执行”的指令周期（指挥中心）。

    3. 存储器：存储程序和数据，分内存（RAM，速度快、断电失数据）和外存（硬盘/U盘，容量大、断电保数据）。

    4. 输入设备：向计算机输入信息（键盘、鼠标、扫描仪）。

    5. 输出设备：输出处理结果（显示器、打印机、音箱）。

    6. 核心：“存储程序”思想——程序和数据预先存入存储器，CPU按顺序执行指令。

    5. TCP/IP协议的分层结构（四层模型）

    1. 网络接口层（链路层）：底层，负责物理介质连接和数据帧传输（如以太网、PPP协议），处理MAC地址。

    2. 网络层（互联层）：负责跨网络路由选择和数据转发，核心协议是IP协议（处理IP地址），辅助协议有ICMP（网络诊断）、ARP（MAC地址解析）。

    3. 传输层：端到端可靠传输，提供流量控制和差错控制，核心协议：

      • TCP：面向连接、可靠（如网页访问、文件传输）；

      • UDP：无连接、快速（如视频通话、游戏）。

    4. 应用层：为应用程序提供网络服务，协议如HTTP（网页访问）、FTP（文件传输）、SMTP（邮件）、DNS（域名解析）。

    五、问答题（每小题10分，共20分）

    1. 以a=10;a=a+1为例，解释程序执行过程

    核心逻辑：程序执行是“存储程序+指令周期”的循环，涉及内存、CPU寄存器、总线协作。

    步骤分解：

    1. 编译阶段：高级语言代码被编译器翻译成机器语言（二进制指令），存储在硬盘可执行文件中。

    2. 加载阶段：操作系统将文件中的指令和数据加载到内存，为变量a分配内存地址（假设为0x1000）。

    3. 执行 a=10：

      • 取指：程序计数器（PC）指向a=10的指令地址，控制器从内存取出指令存入指令寄存器（IR）。

      • 译码：译码器解析指令为“将立即数10存入0x1000”。

      • 执行：运算器将10送入通用寄存器（AX），通过地址总线找到0x1000，数据总线写入10（内存0x1000=10）。

    4. 执行 a=a+1：

      • 取指：PC递增，取出a=a+1的指令存入IR。

      • 译码：解析为“从0x1000取数→加1→存回0x1000”。

      • 执行：① 从0x1000读出10存入AX；② AX+1=11；③ 写回0x1000（a=11）。

    5. 结束：PC指向下一步指令，无指令则程序终止。

    汇编语言示例（x86）：

    mov dword ptr [a], 0Ah   ; a=10，将10存入a的内存地址
mov eax, dword ptr [a]    ; 取出a的值到eax
add eax, 1                ; eax=eax+1
mov dword ptr [a], eax    ; 存回a的内存地址

    2. Web访问的步骤（以访问www.baidu.com为例）

    核心流程：域名解析→建立连接→发送请求→接收响应→渲染页面

    1. 输入URL并回车：浏览器接收域名www.baidu.com，需先解析为IP地址。

    2. DNS域名解析：

      • 浏览器查询本地DNS缓存→无则查本地DNS服务器（路由器分配）→无则向上级DNS（ISP）→根DNS→.com顶级域DNS→百度权威DNS。

      • 权威DNS返回IP地址（如180.101.49.11），传回浏览器。

    3. 建立TCP连接（三次握手）：

      • 客户端→服务器：SYN（请求连接）；

      • 服务器→客户端：SYN+ACK（确认连接）；

      • 客户端→服务器：ACK（连接建立）。

    4. 发送HTTP请求：客户端通过TCP连接发送请求报文（如GET / HTTP/1.1），包含请求方法、URL、浏览器信息等。

    5. 服务器响应：百度服务器解析请求，返回HTTP响应报文（状态码200 OK，响应体为首页HTML）。

    6. 关闭TCP连接（四次挥手）：客户端→FIN→服务器→ACK+FIN→客户端→ACK→连接关闭。

    7. 浏览器渲染页面：

      • 解析HTML构建DOM树，解析CSS构建CSSOM树，结合生成渲染树。

      • 布局（计算元素位置）→绘制（显示像素），加载图片/JS等资源，最终显示百度首页。

    1. 递归求n!（补全代码）

    def fac(n):
    if n == 0 or n == 1:  # 边界条件：0!和1!都等于1
        r = 1
    else:
        r = n * fac(n-1)  # 递归公式：n! = n*(n-1)!
    return r

m = 4
print(fac(m))  # 输出：24（4! = 4*3*2*1=24）

    解析：递归的核心是“分解问题+边界条件”，避免无限递归（需明确终止条件）。

    2. 二分法求实数平方根（补全代码）

    def square_root(n):
    i = 0
    m_max = n
    m_min = 0
    if n < 1:  # 处理n<1的情况（如0.25），平方根在[n,1]之间
        m_max = 1.0
    g = (m_min + m_max) / 2  # 初始中间值（二分法核心：取上下界中点）
    while abs(g*g - n) > 0.00000000001:  # 误差阈值（满足精度要求）
        if g*g < n:
            m_min = g  # 平方根在[g, m_max]，更新下界
        else:
            m_max = g  # 平方根在[m_min, g]，更新上界
        g = (m_min + m_max) / 2  # 重新计算中点（逼近真实值）
        i = i + 1
        print("%d:%.13f" % (i, g))

# 函数之外执行
n = float(input("请输入一个实数："))
square_root(n)

    示例运行：输入4，输出逐步逼近2.0；输入2，逼近1.4142135623731（√2的近似值）。

    解析：二分法通过不断缩小上下界范围，快速逼近真实平方根，时间复杂度O(log n)，精度可通过阈值调整。
