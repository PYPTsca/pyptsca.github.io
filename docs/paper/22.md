## 四、计算题（每小题3分，共15分）

### 1. 求十进制数（-3）的8位补码

#### 解题步骤：

1. **求绝对值的原码**：3的二进制为`11`，8位原码（正数符号位为0）为`00000011`。

2. **求负数的原码**：符号位改为1（表示负数），其余位不变，得`10000011`。

3. **求反码**：原码除符号位外，其余位按位取反，得`11111100`。

4. **求补码**：反码加1，得`11111101`。

#### 答案：`11111101`

### 2. 无符号二进制数(0011011011110001)₂转换成十六进制

#### 解题步骤：

1. **二进制分组**：从右往左每4位为一组（不足4位左侧补0，不影响值），原二进制分组为：`0011 0110 1111 0001`。

2. **分组转十六进制**：每4位二进制对应1位十六进制：0011=3，0110=6，1111=F，0001=1。

3. **组合结果**：将各组结果拼接，得`36F1`。

#### 答案：`36F1h`（或`36F1`）

### 3. 十进制数23.6转换为二进制（精确到小数点后4位）

#### 解题步骤：

1. **整数部分（23）转换**：除以2取余，从下往上排列：23÷2=11余1 → 11÷2=5余1 → 5÷2=2余1 → 2÷2=1余0 → 1÷2=0余1 → 整数部分`10111`。

2. **小数部分（0.6）转换**：乘2取整，保留4位小数：0.6×2=1.2（取整1）→ 0.2×2=0.4（取整0）→ 0.4×2=0.8（取整0）→ 0.8×2=1.6（取整1）→ 小数部分`1001`。

3. **组合结果**：整数部分+小数部分，得`10111.1001`。

#### 答案：`10111.1001`

### 4. 寄存器指令执行结果（十进制）

#### 解题步骤：

1. **指令解析**：  

    - `mov R2, R1`：将R1的值（10h，十六进制=十进制16）赋值给R2，此时R2=16。

    - `add R2, R2, 01h`：R2 = 16 + 1（01h=十进制1）=17。

    - `store (500), R2`：将R2的值（17）存入主存地址500处。

2. **结果转换**：题目要求转换为十进制，直接得17。

#### 答案：`17`

### 5. 计算表达式`11//5%2`的值

#### 解题步骤：

1. **运算优先级**：`//`（整除）和`%`（取余）同级，从左到右计算。

2. **第一步计算整除**：11//5=2（整除取商的整数部分）。

3. **第二步计算取余**：2%2=0（2除以2余数为0）。

#### 答案：`0`

## 五、简答题（每小题4分，共20分）

### 1. 多道进程系统中进程状态转换（含图）

#### （1）进程三状态模型

进程有三种核心状态：**运行态**、**就绪态**、**阻塞态**。

#### （2）状态转换及原因

1. 就绪态 → 运行态：CPU调度（操作系统从就绪队列选择一个进程分配CPU）。

2. 运行态 → 就绪态：时间片用完（进程占用CPU的时间到）或有更高优先级进程进入就绪态。

3. 运行态 → 阻塞态：进程请求资源（如I/O操作、等待信号量），需暂停等待。

4. 阻塞态 → 就绪态：资源到位（如I/O完成、等待的信号量释放），进程被唤醒。

#### （3）状态转换图

```Plain Text
┌───────────┐       CPU调度       ┌───────────┐
│  就绪态   ├─────────────────────→│  运行态   │
└───────────┘                     └───────────┘
        ↑                             ↓
        │ 资源到位（唤醒）             │ 请求资源（等待）
        └─────────────────────────────┘
                ┌───────────┐
                │  阻塞态   │
                └───────────┘
```

### 2. 以`n=n+1`为例，程序执行过程

#### 核心逻辑：程序执行是“存储程序+指令周期（取指-译码-执行）”的循环，涉及内存、CPU、总线协作。

#### 步骤分解：

1. **编译阶段**：高级语言代码`n=n+1`被编译器翻译成机器语言指令（二进制），存储在硬盘可执行文件中。

2. **加载阶段**：操作系统将文件中的指令和变量`n`的数据加载到内存，为`n`分配内存地址（假设为`0x2000`）。

3. **执行指令（指令周期）**：

    - 取指：程序计数器（PC）指向`n=n+1`的指令地址，控制器从内存取出指令存入指令寄存器（IR）。

    - 译码：译码器解析指令为“从`0x2000`取`n`的值→加1→存回`0x2000`”。

    - 执行：① ALU从内存`0x2000`读出`n`的值，存入通用寄存器；② ALU执行“加1”运算；③ 结果通过数据总线写回内存`0x2000`，`n`的值更新。

4. **结束**：PC指向下一步指令，完成执行。

#### 汇编语言示例（简化）：

```
mov eax, [0x2000]  ; 从内存0x2000取n的值到eax
add eax, 1         ; eax = eax + 1
mov [0x2000], eax  ; 结果存回内存0x2000
```

### 3. 递归函数及编程基本步骤

#### （1）递归函数定义

递归函数是**在函数体内直接或间接调用自身**的函数，核心是“将复杂问题分解为规模更小的同类问题”。

#### （2）编程基本步骤

1. **确定边界条件（终止条件）**：明确递归何时停止（避免无限递归），如“列表长度为1时，最小值就是自身”。

2. **确定递归公式（分解问题）**：将原问题分解为规模更小的同类问题，如“列表最小值=min(第一个元素，剩余列表的最小值)”。

3. **实现递归调用**：在函数体内调用自身处理子问题，传入规模更小的参数（如剩余列表）。

4. **返回结果**：将子问题的结果与当前处理结果结合，返回最终答案。

### 4. TCP/IP协议中传输层的重要性

传输层是TCP/IP模型的核心中间层，连接应用层和网络层，重要性体现在：

1. **端到端通信保障**：为应用程序提供进程级的端到端通信（如浏览器与网页服务器的进程通信），屏蔽底层网络层的路由、转发细节。

2. **可靠性与效率选择**：提供两种核心协议：

    - TCP：面向连接、可靠传输（差错重传、流量控制、拥塞控制），保障数据完整有序（如文件传输、网页访问）。

    - UDP：无连接、高效传输，牺牲可靠性换取速度（如视频通话、游戏）。

3. **隔离上下层**：应用层无需关注数据传输细节（如如何处理丢包），网络层无需关注应用需求（如是否需要可靠传输），降低系统耦合度。

4. **端口寻址**：通过端口号区分同一主机上的不同应用程序（如HTTP用80端口，HTTPS用443端口），实现多应用并发通信。

## 六、程序填空题（每小题5分，共10分）

### 1. 递归查找列表最小值（补全代码）

```Python
def M(L):
    if len(L) == 1:  # 边界条件：列表长度为1时，最小值是唯一元素
        return L[0]
    return min(L[0], M(L[1:]))  # 递归公式：最小值=min(第一个元素，剩余列表的最小值)

# 函数外代码
L = [5, 6, 1, 9, 2]
print(M(L))  # 输出：1
```

#### 解析：

- 边界条件：列表长度为1时，直接返回该元素（无需递归）。

- 递归调用：每次取列表第一个元素，与剩余列表（L[1:]）的最小值比较，返回较小值，逐步缩小问题规模。

### 2. 二分法查找升序列表中的整数（补全代码）

```Python
def find(L, x):  # L为升序列表，x为目标元素
    max_idx = len(L) - 1  # 修正变量名（避免与内置函数max冲突）
    min_idx = 0
    while min_idx <= max_idx:  # 修正循环条件（覆盖最后一个元素）
        m = (min_idx + max_idx) // 2  # 计算中间索引
        if L[m] == x:  # 找到目标元素，返回索引
            return m
        if x > L[m]:  # 目标元素在右半部分，更新左边界
            min_idx = m + 1
        elif x < L[m]:  # 目标元素在左半部分，更新右边界
            max_idx = m - 1

# 函数外代码
print(find([34, 56, 58, 67, 88, 99], 56))  # 输出：1
```

#### 解析：
