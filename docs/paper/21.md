## 四、计算题（每小题2分，共10分）

### 1. 求十进制数（-2）的8位补码

#### 解题步骤：

1. 求绝对值的原码：2的二进制为`10`，8位原码为`00000010`（符号位0表示正数）。

2. 求负数的原码：符号位改为1，得`10000010`。

3. 求补码：负数补码=原码除符号位外取反加1  

    - 除符号位取反：`0000010`→`1111101`  

    - 加1→`1111110`  

    - 加符号位→最终补码`11111110`。

#### 答案：`11111110`

### 2. 二进制数(1010110110011110)₂转换成十六进制

#### 解题步骤：

1. 二进制转十六进制：从右往左每4位分组，不足4位补0（左侧补0不影响值）：原二进制：`1010 1101 1001 1110`（已整除4组）。

2. 每组4位二进制转十六进制：1010=A，1101=D，1001=9，1110=E → 组合为`A D 9 E`。

#### 答案：`ADEh`（或`ADE`）

### 3. IP地址二进制转点分十进制，并判断公网/私网

#### 解题步骤：

1. 二进制分组转换：将32位二进制按8位拆分4段，每段转十进制：  

    - 11000000→192，10101000→168，00000001→1，00001010→10。

2. 组合为点分十进制：`192.168.1.10`。

3. 私网地址范围：[192.168.0.0](192.168.0.0)~[192.168.255.255](192.168.255.255)（属于C类私网地址），故为私网地址。

#### 答案：点分十进制`192.168.1.10`，私网地址

### 4. 寄存器指令执行结果计算

#### 解题步骤：

1. 指令分析：  

    - `mov R2, R1`：将R1的值（14h，十六进制=十进制20）赋值给R2（R2=14h）。

    - `add R2, R2, 0Ah`：R2=14h+0Ah=1Eh（十六进制，14h=20，0Ah=10，20+10=30=1Eh）。

    - `store (800), R2`：将R2的值（1Eh）存入主存800地址。

#### 答案：主存800处存储结果为`1Eh`（或十进制30）

### 5. 计算表达式`8//5%2`的值

#### 解题步骤：

1. 运算优先级：`//`（整除）和`%`（取余）同级，从左到右计算。

2. 第一步：8//5=1（整除取商的整数部分）。

3. 第二步：1%2=1（1除以2余数为1）。

#### 答案：`1`

## 五、简答题（每小题4分，共20分）

### 1. 现代操作系统的中断类型及举例

#### （1）硬件中断（外部中断）

- 定义：由计算机外部硬件设备触发，向CPU请求处理的中断。

- 举例：键盘输入中断（按键盘键）、鼠标点击中断、磁盘I/O中断（磁盘读写完成）、时钟中断（定时触发任务）。

#### （2）软件中断（内部中断/异常）

- 定义：由程序执行错误或主动请求触发的中断。

- 举例：除0异常、内存访问越界、系统调用（如Python中`print`底层调用操作系统接口）、断点中断（调试程序时）。

### 2. CPU的核心部件及作用

1. **算术逻辑单元（ALU）**：核心运算部件，负责执行算术运算（加减乘除）和逻辑运算（与、或、非、比较）。

2. **控制器（CU）**：指挥中心，协调CPU各部件及计算机其他设备工作，执行“取指-译码-执行”的指令周期。

3. **寄存器组**：高速存储单元，用于临时存放指令、数据和地址，分为通用寄存器（如累加器，灵活存储）和专用寄存器（如程序计数器PC、指令寄存器IR，功能固定）。

### 3. 算法的五个重要特征

1. **有穷性**：算法执行有限步骤后必须终止，不能无限循环。

2. **确定性**：每一步操作无歧义，给定相同输入，必产生相同输出。

3. **可行性**：每一步操作都能通过基本运算（如加减乘除）在有限次内完成。

4. **输入**：0个或多个输入（0个输入指初始条件固定）。

5. **输出**：1个或多个输出（输出是问题的解，与输入对应）。

### 4. TCP/IP（IPv4）协议分层及作用（四层模型）

1. **网络接口层（链路层）**：底层，负责物理介质连接和数据帧传输（如以太网协议），处理MAC地址和数据链路错误。

2. **网络层（互联层）**：核心层，负责跨网络路由选择和数据转发，核心协议是IP协议（处理IP地址），辅助协议有ICMP（网络诊断）、ARP（MAC地址解析）。

3. **传输层**：端到端通信层，提供可靠/高效传输，核心协议：  

    - TCP：面向连接、可靠传输（如网页访问、文件传输），提供流量控制和差错恢复；  

    - UDP：无连接、快速传输（如视频通话、游戏），无差错恢复。

4. **应用层**：为应用程序提供网络服务，协议如HTTP（网页访问）、FTP（文件传输）、SMTP（邮件发送）、DNS（域名解析）。

### 5. 网页访问流程（以访问`www.baidu.com`为例）

1. **输入URL并回车**：浏览器接收域名`www.baidu.com`，需先解析为IP地址。

2. **DNS域名解析**：浏览器依次查询本地DNS缓存→本地DNS服务器→上级DNS服务器→根DNS→百度权威DNS，最终获取IP地址（如`180.101.49.11`）。

3. **建立TCP连接（三次握手）**：客户端与百度服务器通过三次握手建立可靠连接（请求→确认→应答）。

4. **发送HTTP请求**：客户端通过TCP连接发送HTTP请求报文（如`GET / HTTP/1.1`），包含请求方法、URL、浏览器信息等。

5. **服务器响应**：百度服务器解析请求，返回HTTP响应报文（状态码200 OK，响应体为首页HTML代码）。

6. **关闭TCP连接（四次挥手）**：数据传输完成后，双方通过四次挥手关闭连接。

7. **浏览器渲染页面**：浏览器解析HTML构建DOM树，解析CSS构建CSSOM树，结合生成渲染树，最终绘制页面并显示。

### 1. 二分法求实数平方根（补全代码）

```Python
def square_root(n):
    i = 0
    m_max = n
    m_min = 0
    if n < 1:  # 处理n<1的情况（如0.25），平方根范围在[n,1]
        m_max = 1.0
    g = (m_min + m_max) / 2  # 初始中间值（二分法核心：取上下界中点）
    while abs(g*g - n) > 0.00000000001:  # 精度阈值
        if g*g < n:
            m_min = g  # 平方根在[g, m_max]，更新下界
        else:
            m_max = g  # 平方根在[m_min, g]，更新上界
        g = (m_min + m_max) / 2  # 重新计算中点（逼近真实值）
        i = i + 1
        print("%d:%.13f" % (i, g))

# 函数之外执行
n = float(input("请输入一个实数："))
square_root(n)
```

#### 解析：二分法通过不断缩小上下界范围逼近平方根，核心是“中点更新+阈值判断”，需处理n<1的边界情况（避免下界大于上界）。

### 2. 递归求汉诺塔移动次数（补全代码）

```Python
def f(n):
    if n == 1:  # 边界条件：1个圆盘只需移动1次
        r = 1
    else:
        r = 2 * f(n-1) + 1  # 递推公式：n个圆盘=2*(n-1个圆盘次数)+1（最底层圆盘移动1次）
    return r

m = int(input("请输入圆盘个数："))
print(f(m))
```

#### 解析：汉诺塔递归核心是“分解问题”，n个圆盘需先将n-1个移到中间柱，再移最底层，最后将n-1个移到目标柱，故次数满足递推公式`f(n)=2f(n-1)+1`。
